## Overview

Ronix is a Roblox script executor made for Windows users who want a simple way to launch Lua scripts in the Roblox client. The emphasis is on ease of use and dependable behavior, with a one-click injection flow that attaches to the Roblox process without manual PID entry or outside keys. Its lightweight design keeps resource usage low, which makes it a practical option for machines with modest hardware while still keeping execution steady.

A built-in Script Hub brings together more than 500 scripts from different game categories, so users can search, preview, and run scripts from the same interface. Ronix also updates automatically to stay aligned with Roblox patches, and its SQLite-backed queue system preserves script preferences and run history between sessions. For game automation or custom Lua testing, it offers a stable setup on current Windows installations.

## Core Capabilities

- **One-Click Injection** - Connect to the Roblox process right away with a single action, without selecting a PID by hand.

- **Batch Execution Mode** - Place several scripts in a queue and run them one after another or at the same time, with timing control.

## FAQ

**Is Ronix safe to use?**  
Ronix is distributed as-is. You are responsible for following Roblox's Terms of Service. It does not include malware, but you should use it at your own discretion.

**Does it work with the latest Roblox update?**  
Yes. The auto-update engine is intended to keep Ronix working with current Roblox versions. When a breakage happens, a patch is typically released within 24 hours.

**How is Ronix different from other free executors?**  
It combines a no-key flow, a built-in debugger, and a persistent queue. Those are capabilities often associated with paid tools.

**Can my Roblox account get banned?**  
Any third-party executor carries risk. Ronix cannot promise account safety. If you are worried, use an alternate account.

**Where are my scripts saved?**  
Custom scripts live in the `scripts/custom` folder. Script hub data is cached under `scripts/hub`. Nothing is sent to external servers.
